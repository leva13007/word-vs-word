# Word vs Word

Живий каталог пар (і груп) близьких за значенням слів, у якому пояснюємо, як їх відрізняти в інженерних контекстах.

## Для кого цей проєкт

- інженери, які читають і пишуть технічну документацію англійською
- розробники, яким важливо точно формулювати причини, наслідки та рішення
- люди, які думають системно, а не “словниково”

## Як організовано

- кожен файл `NNNN-keyword.md` присвячений одному кластеру дієслів/іменників;
- внутрішній номер (`NNNN`) відповідає порядку публікації й допомагає лінкувати приклади між собою;
- у текстах зберігаємо єдиний формат: вступ → сигнал для інженера → детальні пояснення по кожному слову → порівняльні таблиці → сценарії та типові помилки.

## Як додавати нові сторінки

1. Обрати близькі слова, які часто плутають (бажано 3‑5 одиниць).
2. Завести новий файл `NNNN-topic.md`, де `NNNN` — наступний вільний номер із провідними нулями.
3. Викласти матеріал у стандартній структурі (див. шаблон у `0001-cause.md`).
4. Додати запис у таблицю нижче: коротку назву, зміст і статус.
